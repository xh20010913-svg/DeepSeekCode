GUIDE

把 DeepSeekCode 放进真实项目目录里运行。

这份指南面向使用和验收：如何安装、配置模型、控制权限、绑定微信、启用 skills、多 Agent、查看长任务状态，以及发布前如何确认能力没有被夸大。

01 安装启动 02 模型配置 03 权限 04 微信远程 05 产物回传 06 Skills / MCP 07 多 Agent 08 测试发布
01

全局安装

推荐用 npm 全局安装。进入任意项目目录后输入 deepseekcode，当前目录就是项目根目录，运行状态写入该目录的 .deepseekcode。

npm install -g @xh12312/deepseekcode --registry https://registry.npmjs.org/
cd /d D:\code\DeepSeekTest
deepseekcode --model deepseek-v4-flash

源码开发时使用：

cd /d D:\code\DeepSeekCode\.release
npm install
npm run build
npm run start -- --project "D:\code\DeepSeekTest" --model deepseek-v4-flash
02

模型配置

日常测试建议使用 flash。需要更强规划时，在 TUI 里用 /model pro 切换。

DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_API_KEY=your_deepseek_api_key
DEEPSEEK_MODEL=deepseek-v4-flash
DEEPSEEKCODE_LANGUAGE=zh-CN
03

权限不是关键词判断

DeepSeekCode 不通过“网页、shell、PPT、代码”等关键词硬判定任务。模型通过 native tool calls 请求工具，runtime 在工具执行前统一做权限 gate。

/shell on
/shell off
/browser on
/permissions

shell 关闭时，TUI 会弹出选择框；个人微信端使用数字审批：1 允许一次，2 本会话允许，3 拒绝，4 停止任务。

04

微信远程控制

推荐先在电脑项目目录启动 TUI，再用 /remote-control 绑定微信。电脑保持运行，微信只做简洁控制、状态查看、权限审批和结果回传。

/remote-control
/remote-control wechat login
/remote-control wechat start
/remote-control wecom start

纯远程模式：

deepseekcode --wechat-login --project "D:\code\DeepSeekTest"
deepseekcode --wechat --project "D:\code\DeepSeekTest" --model deepseek-v4-flash
deepseekcode --wecom --project "D:\code\DeepSeekTest" --model deepseek-v4-flash

微信端常用命令：

/run 继续完成这个项目
/status
/status full
/ask 现在做到哪了
/artifacts
/stop
05

远程产物回传

产物回传由 runtime 根据真实文件类型规划，不靠 prompt 关键词。微信端优先看图片预览和简洁摘要，不刷屏发送所有源码文件。

HTML优先浏览器截图，再给入口文件摘要。
DOCX / PPTX / XLSX发送微信可打开的原文件。
PDF发送 PDF，未来可补前几页图片预览。
图片直接作为图片发送。
MD / TXT默认发送短摘要，需要时再发文件。
多文件项目发送摘要、入口、截图和 manifest，不逐个发送 CSS/JS。
06

Skills、Plugins 和 MCP

安装器支持本地路径、GitHub、Git URL 和 file:// Git 源；源里有多个 SKILL.md 时，不带名称会批量安装全部 skill。模型会根据 skill description 通过 search_skills / invoke_skill 自动选择，不依赖写死关键词。

/skills install greensock/gsap-skills
/skills install-all greensock/gsap-skills
/skills search gsap
/skills run gsap-core "给当前网页加动画"
/plugins
/mcp
07

多 Agent 和旁路问答

主 agent 可以通过 native workflow tools 创建角色、共享黑板消息和 Reviewer 验收角色。用户没有分配角色时，模型会按项目自动设计角色。该能力仍标为实验中。

开启多 agent 协作，让前端、测试和验收一起完成这个页面。
/ask 现在做到哪了？
/ask 生成了哪些文件？
08

测试和发布

真实测试产物只放在独立目录，例如 D:\code\DeepSeekTest。发布只从 D:\code\DeepSeekCode\.release 提交。

npm.cmd run typecheck
npm.cmd run build
npm.cmd pack --dry-run
npm.cmd pack --pack-destination D:\code\DeepSeekTest\npm-packages

不提交测试产物、prompt audit、运行数据库、登录态、node_modules 或临时 npm tarball。
