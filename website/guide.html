GUIDE

把 DeepSeekCode 放进真实项目目录里运行。

这份指南面向使用和验收：如何安装、配置模型、控制权限、绑定微信、开启多 Agent、查看长任务状态，以及发布前如何确认能力没有被夸大。

01 安装启动 02 模型配置 03 权限 04 微信远程 05 多 Agent 06 产物回传 07 Skills / MCP 08 测试 09 发布
01

全局安装

推荐用 npm 全局安装。进入任意项目目录后输入 deepseekcode，当前目录就是项目根目录，运行状态写入该目录的 .deepseekcode。

npm install -g @xh12312/deepseekcode --registry https://registry.npmjs.org/
cd D:\code\DeepSeekTest
deepseekcode

源码开发时使用：

git clone https://github.com/xh20010913-svg/DeepSeekCode.git
cd DeepSeekCode
npm install
npm run build
npm run start -- --project "D:\code\DeepSeekTest"
02

模型配置

最小配置是 DeepSeek 兼容端点、API key 和模型名。测试时建议使用 flash；需要更强推理时在 TUI 里用 /model pro 切换。

DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_API_KEY=your_deepseek_api_key
DEEPSEEK_MODEL=deepseek-v4-flash
DEEPSEEKCODE_LANGUAGE=zh-CN
03

权限不是关键词判断

DeepSeekCode 不通过“网页、shell、代码”等关键词硬判定任务。模型通过 native tool calls 请求工具，runtime 在工具执行前统一做权限 gate。

deepseekcode --permission-profile safe
deepseekcode --permission-profile dev
/shell on
/permissions

shell 关闭时，TUI 会弹出选择框；微信端会用数字菜单审批：1 允许一次，2 本会话允许，3 拒绝，4 停止任务。

04

微信远程控制

第一版保留两个通道：企业微信 WeCom 和个人微信 OpenClaw。电脑需要保持运行，微信只做简洁控制、状态查看、权限审批和产物预览。

deepseekcode --wecom --project "D:\code\DeepSeekTest" --model deepseek-v4-flash
deepseekcode --wechat-login --project "D:\code\DeepSeekTest"
deepseekcode --wechat --project "D:\code\DeepSeekTest" --model deepseek-v4-flash

TUI 内推荐使用：

/remote-control
/remote-control wechat login
/remote-control wechat start
/remote-control wecom start

微信端常用命令：

/run 继续完成这个项目
/status
/status full
/ask 现在做到哪了
/artifacts
/stop
05

多 Agent 工作流

v0.2.7 开始，模型可以通过 native workflow tools 启动多 Agent 协作。用户可以指定角色，也可以让主 agent 根据项目设计角色和验收标准。

开启多 agent 协作：前端负责页面，测试负责验收，Reviewer 检查是否真的生成入口文件。
start_agent_workflow创建角色、目标、共享黑板和验收约束。
send_agent_message记录角色之间的消息，避免上下文串到别的项目。
agent_status查看角色进度、问题、产物和 Reviewer 状态。
finish_agent_workflow关闭工作流并记录验收结论。

这仍是实验能力，还不是完全独立后台 worker pool。

06

远程产物回传

产物回传由 runtime 根据真实文件类型规划，不靠用户 prompt 关键词。微信端优先看图片预览和简洁摘要，不刷屏发送所有源码文件。

HTML优先生成浏览器截图，再给入口文件摘要。
DOCX / PPTX / XLSX发送原文件；本机可转换时附加 PDF 或首屏预览。
PDF发送 PDF，并可附前几页图片。
图片直接按图片发送。
MD / TXT默认发短摘要，用户要求时再发文件。
多文件项目发摘要、入口、manifest 和截图，不逐个发送 CSS/JS。
07

Skills、Plugins 和 MCP

Skills 可以被主 agent 和子 agent 调用；MCP 当前通过统一 mcp_call 入口接入。安装器支持本地目录、GitHub、Git URL 和 file:// Git 源；如果源里有多个 SKILL.md，不带名称会批量安装全部 skill。模型会根据 skill description 通过 search_skills / invoke_skill 自主选择，不依赖写死的任务关键词。

/skills
/skills install "D:\skills\office-report"
/skills install greensock/gsap-skills
/skills install-all greensock/gsap-skills
/skills install greensock/gsap-skills gsap-core
/skills search gsap
/plugins
/mcp
08

真实测试

测试产物只放在独立目录，例如 D:\code\DeepSeekTest。真实验收应覆盖网页项目、DOCX/PPTX、失败后自修复、多 Agent、微信远程和浏览器截图。

npm.cmd run typecheck
npm.cmd run build
cd /d D:\code\DeepSeekTest
deepseekcode --model deepseek-v4-flash
09

发布边界

发布只从 D:\code\DeepSeekCode\.release 提交。不提交 D:\code\DeepSeekTest、.env、prompt audit、测试报告、node_modules、登录态或 runtime 数据库。

git status --short
npm.cmd run typecheck
npm.cmd run build
