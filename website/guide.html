DeepSeekCode 使用指南

面向真实本地开发：启动、模型切换、native tool calling、权限选择、Skills/Plugins、上下文缓存和发布边界。

01 安装启动 02 工具链路 03 权限交互 04 上下文与缓存 05 Skills / Plugins 06 发布规则
01

安装启动

建议先在独立测试目录运行，不把测试产物写进项目源码目录。

git clone https://github.com/xh20010913-svg/DeepSeekCode.git
cd DeepSeekCode
npm install
npm run build
npm run start -- --project "D:\code\DeepSeekTest"

如需切换模型，可在 TUI 中使用 /model 选择，或直接输入：

/model flash
/model pro
02

Native Tool Calling

v0.2 之后主链路是 DeepSeek tools → tool_calls → 本地工具 → tool result → 下一轮。本地仍使用 Zod schema 和 tool registry 做参数校验、路径安全、权限和结果压缩。

已验证文件读写、patch、glob/grep/list、todo、DOCX/PPTX runtime、skills/plugins 基础路径、TencentDB-Agent-Memory。
部分可用Browser CDP、MCP stdio、Office 质量增强、多 agent 调度。
实验中PDF 生成、长任务 worker pool、真实场景自动评测。
保留computer_use 还没有完整 GUI 桥接，不宣传成可用能力。
03

权限交互

模型可以请求 run_command，但 shell 关闭时不会直接失败。runtime 会在执行前拦截，TUI 弹出可选择的权限 gate：允许一次、本会话允许、拒绝或取消。

/shell on
/shell off
/permissions

Plan gate 和 AskUserQuestion 也使用同一套选择控件；方向键移动，Enter 确认，Esc 取消。

04

上下文与缓存

DeepSeekCode 不依赖无脑回放全量聊天历史。它把上下文分成稳定前缀、最近对话、rolling summary、runtime_run_state 和 tool_result micro summary，以提高缓存复用并降低长任务成本。

/cache plan "做一个大型网页项目"
/usage
/cost
05

Skills / Plugins

DeepSeekCode 支持把本地或 Git 来源的 skill/plugin 安装到 .deepseekcode 目录，并兼容部分 .claude skill 结构。plugin 内的 skill 可被发现和调用。

/skills
/skills install <path-or-git-url>
/plugins
/plugins install <path-or-git-url>
06

发布边界

GitHub 发布只从 D:\code\DeepSeekCode\.release 提交。测试目录、prompt audit、场景报告、.env、node_modules 和开发草稿都不属于发布内容。

npm run typecheck
npm run build
git status --short
git push origin main

公开 README 和网站只写真实接通能力；未完成项会标成部分可用、实验中或保留。
