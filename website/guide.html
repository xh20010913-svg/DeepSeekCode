GUIDE

把 DeepSeekCode 放进任何项目目录运行

这是一份面向使用和部署的说明：从 npm 安装、模型配置、权限选择，到企业微信 / 个人微信远程控制、工具能力边界和发布前检查。

01 安装启动 02 配置模型 03 权限交互 04 微信远程 05 工具链路 06 上下文缓存 07 Skills / Plugins 08 真实测试 09 发布边界
01

安装后在任意项目目录启动

推荐通过 npm 全局安装。进入项目目录后输入 deepseekcode，当前目录就是项目根目录，运行数据写入该目录的 .deepseekcode。

npm install -g @xh12312/deepseekcode --registry https://registry.npmjs.org/
cd D:\code\DeepSeekTest
deepseekcode

源码 checkout 只用于开发和测试：

git clone https://github.com/xh20010913-svg/DeepSeekCode.git
cd DeepSeekCode
npm install
npm run build
npm run start -- --project "D:\code\DeepSeekTest"
02

模型和运行目录

最小配置是 DeepSeek 兼容端点、API key 和模型名。测试密钥建议只放在测试项目目录的 .env。

DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_API_KEY=your_deepseek_api_key
DEEPSEEK_MODEL=deepseek-v4-flash
DEEPSEEKCODE_LANGUAGE=zh-CN
默认项目未传 --project 时使用当前目录。
默认数据未传 --data-dir 时写入 <project>/.deepseekcode。
模型切换TUI 内使用 /model、/model flash 或 /model pro。
03

权限交互

默认 safe 模式关闭 shell 和 browser。TUI 启动时如果 shell 未开启，会先询问是否为本会话开启 shell 权限；方向键选择，Enter 确认，Esc/N 保持关闭。

deepseekcode --permission-profile safe
deepseekcode --permission-profile dev
deepseekcode --allow-shell

模型真正请求 run_command 时仍然由 runtime permission gate 拦截。用户看到工具名、命令、cwd、风险等级和允许/拒绝选项；这不是通过关键词判断用户任务。

04

微信远程控制

v0.2.6 支持两个实验远程通道：企业微信智能机器人长连接，以及个人微信 OpenClaw 扫码登录和长轮询。电脑端需要保持运行，微信端只显示简洁进度、权限请求和产物摘要。

deepseekcode --wecom --project "D:\code\DeepSeekTest" --model deepseek-v4-flash
deepseekcode --wechat-login --project "D:\code\DeepSeekTest"
deepseekcode --wechat --project "D:\code\DeepSeekTest" --model deepseek-v4-flash

TUI 内可用：

/remote-control
/remote-control wecom start
/remote-control wechat login
/remote-control wechat start
/remote-control wechat stop

个人微信没有模板卡片，因此审批用数字回复：1 允许一次，2 本会话允许，3 拒绝，4 停止任务。个人微信 hook、逆向协议和 wxauto 不在默认版中接入。

DEEPSEEKCODE_WECOM_BOT_ID=your_wecom_bot_id
DEEPSEEKCODE_WECOM_BOT_SECRET=your_wecom_bot_secret
DEEPSEEKCODE_WECHAT_OPENCLAW_ENABLED=true
DEEPSEEKCODE_WECHAT_PROJECT_ROOTS=D:\code\DeepSeekTest;D:\work
05

Native Tool Calling

主链路是 DeepSeek tools -> tool_calls -> local tools -> tool_result -> next turn。不支持 native tool calls 的模型或网关会明确失败，不静默回退旧 JSON planner。

已验证文件读写、patch、glob/grep/list、todo、DOCX/PPTX runtime、skills/plugins 基础路径、TencentDB-Agent-Memory。
部分可用Browser CDP、MCP stdio、Office 质量增强、多 agent 调度。
实验中PDF 生成、微信远程控制、长任务 worker pool、真实场景自动评测。
保留computer_use 和个人微信 hook 还没有完整桥接，不宣传成可用能力。
06

上下文、缓存和记忆

DeepSeekCode 不把所有历史原文无限塞回 prompt。稳定规则和工具 schema 放前面提高缓存复用；旧对话压成 rolling summary；长工具结果压成摘要和 artifact manifest。

/cache
/cache guard <goal>
/usage
/cost
/memory status
/memory search <query>
07

Skills / Plugins

DeepSeekCode 支持项目、用户、插件 skill 发现和调用；兼容 .claude skill，但安装目标写入 .deepseekcode。

/skills
/skills install "D:\skills\office-report"
/skills install https://github.com/example/agent-skills/tree/main/office/report
/plugins
/plugins install "D:\plugins\review-kit"
/plugins validate
08

真实测试

真实测试产物只放在独立目录，例如 D:\code\DeepSeekTest。推荐任务包括大型网页、答辩 PPT、课程 PPT、OFDR 原理 PPT、DOCX 报告、失败后自修复、多 agent 协作和浏览器验证。

cd /d D:\code\DeepSeekTest
deepseekcode --model deepseek-v4-flash
/runs report latest "D:\code\DeepSeekTest"
09

发布边界

发布只从 D:\code\DeepSeekCode\.release 提交。不提交测试目录、prompt audit、测试报告、.env、runtime 数据库、node_modules 或开发草稿。

npm.cmd run typecheck
npm.cmd run build
npm.cmd pack --dry-run
git status --short
