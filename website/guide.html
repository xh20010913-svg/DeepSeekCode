GUIDE

DeepSeekCode 使用指南。

这份指南面向真实使用和验收：如何安装、配置模型、控制权限、绑定微信、使用 skills/MCP、多 Agent、查看长任务状态，以及发布前如何确认能力没有被夸大。

01 安装启动 02 模型配置 03 权限 04 通用验收 05 微信远程 06 产物回传 07 Skills / MCP 08 多 Agent 09 测试发布
01

全局安装

推荐使用 npm scoped package。进入任意项目目录后输入 deepseekcode，当前目录就是项目根目录，运行状态写入该目录的 .deepseekcode。

npm install -g @xh12312/deepseekcode --registry https://registry.npmjs.org/
cd /d D:\code\DeepSeekTest
deepseekcode --model deepseek-v4-flash

源码开发时使用：

cd /d D:\code\DeepSeekCode\.release
npm install
npm run build
npm run start -- --project "D:\code\DeepSeekTest" --model deepseek-v4-flash
02

模型和数据目录

测试密钥建议放在测试目录的 .env 或系统环境变量里。每个项目目录独立保存会话、runs、artifacts、approval gates 和记忆，降低跨项目串上下文的概率。

set DEEPSEEK_API_KEY=your-key
deepseekcode --model deepseek-v4-flash
/model deepseek-v4-pro
03

权限

shell、browser、MCP、远程审批都走 runtime permission gate。模型可以请求工具，但不能绕过权限层。关闭 shell 时，工具请求会进入审批或返回明确反馈。

/shell on开启本次会话 shell 权限。
/browser on开启浏览器/CDP 验证能力。
/permissions查看当前权限 profile。
04

通用任务契约和验收

v0.3 的核心是 TaskCompletionContract 和 verify_task。模型生成目标、预期产物、可验证行为和完成标准；runtime 根据真实文件、项目结构、工具结果和合约选择验证器。

支持的验证方向包括代码项目、CLI 脚本、网页/前端、真实 PDF、DOCX/PPTX/XLSX、Markdown/研究报告、数据任务、图片媒体、skills/plugins、MCP 和自动化任务。网页只是其中一类。

/status full
/artifacts
/cache report
/cache trend
/memory doctor
/agent doctor
/verify explain
/git doctor
/project processes
/project stop latest
/terminal reset
05

微信远程

推荐从 TUI 里启动远程控制，这样桌面和微信共享同一个项目状态。个人微信 OpenClaw 和企业微信 WeCom 都是实验能力。

/remote-control wechat start
/remote-control status
/ask 现在任务做到哪了
/status full
/artifacts

普通聊天应该正常回答；长任务中的 /ask 是旁路只读问答，不应该污染主任务。

06

产物回传

远程端不应该收到整屏源码。runtime 根据真实产物类型生成 RemoteDeliveryPlan。

HTML/UI截图 + 入口摘要。
PDF必须是真实 .pdf；检查页数和结构，能转图时附预览图。
DOCX/PPTX/XLSX微信可预览文件；能转图时附预览图。
Markdown/Text短摘要；用户要求时再发文件。
多文件项目manifest、入口、启动命令、验收结果和关键预览。
07

Skills、Plugins 和 MCP

Skill 是任务提示和工作流，不替代底层工具。模型可以通过 search_skills 和 invoke_skill 自动调用匹配 skill。

/skills install https://github.com/greensock/gsap-skills
/skills search gsap
/skills validate gsap-core
/mcp status
/mcp call demo ping {}
08

多 Agent

自然语言可以触发多 Agent。默认先进入 Planner 计划闸门，用户选择执行、修改、重生成或取消后才继续。固定角色只有 Planner 和 AcceptanceReviewer；中间执行角色由模型按任务契约、产物类型、工具需求和验收风险动态生成。每个角色保留 workflow-local skill、本地 transcript、tool_result 摘要和 checkpoint。AcceptanceReviewer 必须围绕通用任务契约和真实 evidence 验收，不只检查网页。

请开启多 agent 协作完成这个项目。
/agents status
/status full

当前多 Agent 仍按实验能力标注，但角色本地 runner、Pixel observer、阻塞点和验收结论已经接入 runtime 状态。

08A

多 Agent Pixel 面板

新的多 Agent 工作流启动时会自动打开内置 Pixel Agents 只读面板；继续、修复、再完善会复用同一个 run 页面，不反复弹新窗口。DeepSeekCode 后端提供真实 run snapshot、WebSocket/SSE 更新和 agent-trace.jsonl，Pixel Agents 负责展示目标、阶段、角色职责、任务黑板、角色移动、进程/缓存/预算趋势、span、产物、阻塞和验收结论。

/agents dashboard
/agents dashboard share
/agents dashboard tunnel
/agents dashboard trace
/agents dashboard close

微信/企微分享可以配置 DEEPSEEKCODE_AGENT_PANEL_PUBLIC_BASE_URL 指向可信 HTTPS 隧道，也可以用 /agents dashboard tunnel 临时启动 Cloudflare Quick Tunnel。链接只暴露单个 run 页面并带短时 token；拿到 URL 和 token 的人仍能查看只读面板，所以只发给自己的微信，TUI 关闭后页面会显示离线状态。

09

测试和发布

发布前先做基础构建，再做真实任务验证。测试产物只放 D:\code\DeepSeekTest。

npm.cmd run typecheck
npm.cmd run build
npm.cmd pack --dry-run

真实任务建议覆盖：代码项目自修复、GSAP 动画、PDF 项目文档、DOCX/PPTX/XLSX、MCP mock、微信远程、长任务停止、终端恢复和缓存报告。
