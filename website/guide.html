01

Setup

Use Node.js >= 22 and run DeepSeekCode against a separate test project before using it on valuable work.

git clone https://github.com/xh20010913-svg/DeepSeekCode.git
cd DeepSeekCode
npm install
npm run build
npm run start -- --project "D:\code\DeepSeekTest"
02

Session restore

DeepSeekCode stores transcript records under the data directory. New CLI processes can restore useful context without replaying raw history forever.

npm run start -- --project "D:\code\DeepSeekTest" --continue -p "Continue the last task"
npm run start -- --project "D:\code\DeepSeekTest" --resume session_xxx -p "Continue the paused work"

The restored prompt includes recent conversation, compact tool summaries, and durable runtime state when available.

03

Context layers

Long tasks use layered context instead of full transcript replay.

Stable prefixSystem rules, tool schemas, output style, project rules, and cache pins.
Recent conversationThe last high-value turns that explain current intent.
tool_result_summaryFailed commands, important paths, artifacts, and validation results.
runtime_run_stateRecent or paused runs, task DAG, actions, artifacts, gates, and progress checkpoints.
04

Multi-agent workflow

The provider multi-agent mode runs Planner, Builder, Tester, and Reviewer roles over durable task state. Role feedback is compacted before it is passed to the next role.

/multi provider Create a SaaS incident handoff package with code, tests, docs, and an acceptance report.

Each role attempt writes an agent progress checkpoint, and current sessions receive compact tool summaries when session focus is active.

05

Validation

npm run typecheck
npm run build

Live release checks were run in D:\code\DeepSeekTest for cross-process resume and multi-agent workflows. The prompt audit verified that recent_conversation, tool_result_summary, and runtime_run_state reached provider prompts.

06

Release safety

Publish only runtime source, public docs, website files, and reusable assets. Keep secrets, local state, tests, staging folders, and private development notes out of the public repository.
