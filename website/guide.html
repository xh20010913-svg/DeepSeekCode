Start in a separate project directory, use native tool-calling models, and inspect `/doctor` before long tasks.

Recommended workflow

  1. Run `/doctor` to confirm provider, model, native tools, skills/plugins, cache, and permissions.
  2. Use `/permissions` or `--permission-profile dev` when shell validation is required.
  3. Run realistic tasks in a test project, not inside the source tree.
  4. Use `/memory status`, `/memory search`, and `/memory conversation` when validating long-term memory.
  5. Export `/runs report latest "D:\work\agent-test"` after each scenario.
  6. Use `/usage`, `/cost`, and `/cache` to inspect token, cache, and long-context behavior.

Long-term memory

DeepSeekCode vendors TencentDB-Agent-Memory as a local MIT runtime. It recalls durable preferences before model calls, captures successful turns afterward, and stores data under the selected runtime data directory.
