DeepSeekCode Website & Runtime Guide

Setup, cache-first workflow, command reference, validation, and public-upload safety for the DeepSeekCode project.

01 Setup 02 Cache guard 03 Commands 04 Validation 05 Upload safety
01

Setup

Use Node.js >= 22 and run the workbench against a separate test project first.

git clone https://github.com/xh20010913-svg/DeepSeekCode.git
cd DeepSeekCode
npm install
npm run build
npm run start -- --project "D:\code\DeepSeekTest"
02

Cache guard workflow

Before a large build task, prepare reusable context without spending model tokens.

/cache
/cache guard 做一个网页小游戏，包含目录、关卡、动效
/cache prepare 做一个网页小游戏，包含目录、关卡、动效
/cache profile save web-game 做一个网页小游戏，包含目录、关卡、动效

The goal is to keep stable project facts early and current task details late, improving DeepSeek prefix-cache reuse.

03

Commands

/helpGrouped help panel.
/doctorProvider and runtime diagnostics.
/cacheReadiness, shape, guard, prepare, profile.
/approval listInspect pending gates.
/diff gitReview local file changes.
/multi provider <task>Planner → Builder → Tester → Reviewer.
/shell on|offToggle shell tool permission.
/browser on|offToggle browser-open permission.
04

Validation

npm run build
npm run typecheck
npm run doctor

For live provider checks, use deepseek-v4-flash unless you explicitly choose a more expensive model.
