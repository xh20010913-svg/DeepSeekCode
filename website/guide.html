DeepSeekCode 使用指南

DeepSeekCode 应该在独立项目目录中运行。源码仓库只用于构建和发布，真实任务、测试产物、prompt audit、运行数据库和 `.env` 都应放在你的工作目录或测试目录里。 

npm install
npm run build
npm run start -- --project "D:\work\agent-test"
/doctor
/tools
/memory status
/model flash

推荐启动流程

  1. 先运行 `/doctor`，确认 provider、native tool calling、skills/plugins、缓存、权限和运行目录。
  2. 需要 shell 验证时再使用 `--permission-profile dev` 或在 TUI 中授权。
  3. 真实任务只在独立项目目录跑，不在源码仓库里生成测试产物。
  4. 验证长期记忆时使用 `/memory status`、`/memory search` 和 `/memory conversation`。
  5. 每个场景结束后用 `/runs report latest "D:\work\agent-test"` 导出 Markdown/JSON 报告。
  6. 用 `/usage`、`/cost`、`/cache` 检查 token、费用估算和缓存命中情况。

长期记忆

DeepSeekCode 内置 TencentDB-Agent-Memory 的 MIT runtime。它会在模型调用前召回相关长期记忆，并在成功回合后捕获对话、提取结构化记忆。 默认存储是本地 SQLite；embedding 和腾讯云 VectorDB 需要显式配置。 

/memory status
/memory search 中文默认
/memory conversation 继续完善

能力状态

README 和官网统一使用“已验证、需权限、部分可用、实验中、保留”五类状态。`computer_use` 仍是保留能力，PDF 仍是实验能力，长任务后台 worker pool、Browser CDP/GUI 自动化和 Office/PPT 质量增强还在继续完善。
