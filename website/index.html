DeepSeek-first local agent runtime

DeepSeekCode

一个运行在终端里的本地 Agent：native tool calling、本地 typed tools、SQLite 持久状态、TencentDB-Agent-Memory 长期记忆、skills/plugins、缓存费用 telemetry、真实用户场景测试。

开始使用 阅读说明书
native tool calls required no JSON planner fallback TencentDB-Agent-Memory 内置 中文默认

Quickstart

在独立项目目录启动。

先构建源码，再把 Agent 指向你要测试或开发的项目目录。

git clone https://github.com/xh20010913-svg/DeepSeekCode.git
cd DeepSeekCode
npm install
npm run build
npm run start -- --project "D:\work\agent-test"

Long-Term Memory

长期记忆接入真实运行链路。

DeepSeekCode 内置 TencentDB-Agent-Memory 的 MIT runtime。每轮调用前召回相关偏好、决策和项目事实，成功回合结束后捕获对话并提取结构化记忆。默认本地 SQLite；embedding 和 Tencent Cloud VectorDB 需要显式配置。

/memory status
/memory search 中文默认
/memory conversation 继续完善

set DEEPSEEKCODE_TDAI_MEMORY=off

Skills / Plugins

扩展体系写入 .deepseekcode。

支持本地路径、GitHub URL、Git URL、file:// Git 源；兼容读取 .claude skill/plugin，但安装目标是 DeepSeekCode 自己的目录。

/skills install https://github.com/example/agent-skills/tree/main/office/report
/plugins install file:///D:/repos/deepseekcode-plugin.git#main
/skills validate
/plugins validate

Real Scenario Tests

测试要像真实用户一样发任务。

大型网站，多轮继续完善，browser 验证 答辩 PPT、课程 PPT、OFDR 原理 PPT DOCX 项目报告 失败后自修复 Planner/Builder/Tester/Reviewer 多 Agent CLI 重启后 --continue / --resume

Reports

每个真实场景可导出报告。

报告包含模型、token、cache hit/miss、工具次数、产物、失败点和修复建议。

set DEEPSEEKCODE_PROMPT_AUDIT_DIR=D:\work\agent-test\prompt-audit
/runs report latest "D:\work\agent-test"
